# 작업 명세서 및 오토세이브 관리

이 문서는 모먼트 인사이트 개발 작업의 기준 문서입니다.
앞으로 새 기능을 만들거나 기존 기능을 수정할 때는 이 문서에 작업 의도, 실행 내역, 검증 결과를 남기고 개발 완료 시 체크합니다.

## 운영 규칙

- 개발 착수 전: Git 상태, 최근 커밋, 현재 작업명세와 다음 작업을 확인하고 `npm run work:autosave`를 실행합니다.
- 중복 방지: 새 구현 전에 기존 코드, 문서, 테스트를 검색하고 이미 있는 기능은 확장하거나 재사용합니다.
- 충돌 방지: 기존 미커밋 변경과 `admin.html`/`client.html`의 공통 동작을 보존하고 범위가 겹치면 현재 diff를 먼저 확인합니다.
- 작업 항목을 추가하고 상태를 `진행 중`으로 둡니다.
- 개발 완료 후: 작업 항목을 `완료`로 체크하고 검증 결과를 적습니다.
- 대상 테스트와 다른 관점의 2차 검증, `git diff --check`, 최종 diff 확인을 모두 통과한 논리 단위만 로컬 커밋합니다.
- 완료·보류·배포 상태와 근거를 핵심 운영 문서에 동기화합니다.
- `git push`, Vercel 배포, Supabase 배포는 대표님이 명시적으로 지시할 때만 진행합니다.
- 배포 산출물(`dist`, 임시 캐시)은 필요 시 재생성하고 기본적으로 저장소에 남기지 않습니다.
- API 키가 들어가는 `.env.local` 파일은 삭제하거나 커밋하지 않습니다.

### 4대 핵심 조회·추적 기능 잠금

- 다른 기능 작업에서는 `scripts/protected-rank-features.lock.json`의 보호 함수·파일·마이그레이션을 수정하지 않습니다.
- 보호 범위는 `키워드 조회`, `N 상품 순위`, `N 30일 순위`, `N 플레이스 30일 순위`의 운영팀·광고주 진입 함수와 핵심 서버 계약입니다.
- 이 잠금은 개발 소스 변경을 배포 전에 차단하는 품질 게이트이며, 운영 사용자의 신규 키워드 조회·상품 순위 조회·상품/플레이스 추적 등록·갱신은 막지 않습니다. 보호 범위 밖의 새로운 별도 기능 개발도 계속 허용합니다.
- 보호 대상이 바뀌면 `npm run check:rank-feature-lock`이 품질·릴리스 단계에서 즉시 실패하므로 잠금을 제거하거나 검사를 우회하지 않습니다.
- 네 기능의 변경은 대표님의 명시적 승인, 변경 사유 기록, 대상 테스트, `npm run check:rank-feature-lock:self-test`, 전체 `npm run check:release` 통과 후에만 허용합니다.
- 모든 검증이 끝난 의도된 변경만 `node scripts/check-protected-rank-features.mjs --print-current` 결과와 대조해 해시 기준을 갱신합니다.

## 작업 상태 기준

- `대기`: 아직 시작하지 않은 작업

## 프로젝트 고정 기억

앞으로 작업을 이어갈 때 아래 기준은 다시 흔들지 않는다. 새 기능을 추가하더라도 기존 로그인, 권한, 데이터 격리, 순위추적, 보고서 운영 흐름을 깨지 않는 범위에서만 수정한다.

### 계정과 권한

- 총관리자 계정은 `mml93-a01`이다.
- 총관리자는 운영팀 코드 생성, 광고주 직접 생성, 활성 계정 조회, 권한 해제를 담당한다.
- 운영팀은 계정당 광고주 1명만 연결한다.
- 광고주는 운영팀 또는 총관리자가 발급한 광고주 코드로만 접속한다.
- 광고주 화면에는 운영팀 코드, 내부 메모, API 기준, 비밀값, 내부 오류 상세를 노출하지 않는다.
- 로그아웃 기능은 유지하고, 새로고침 시에는 저장된 코드로 서버 재검증 후 세션을 복구한다.

### 현재 사용 가능 도구

- 현재 네이버 상품 단건·30일 기능은 `네이버 쇼핑 검색 API 결과 순번`을 제공한다. 실제 쇼핑 화면의 페이지·위치 순위로 표현하지 않는다.
- Meta 광고 조사는 메뉴와 화면에서 `개발중`으로 표시한다.
- 네이버 플레이스 순위 추적은 상품 순위와 분리된 별도 기능으로 다룬다.
- 외부 API 기준과 내부 계산식은 광고주 화면에 설명하지 않는다.

### 네이버 순위 추적

- 네이버 상품 숫자는 공식 쇼핑 검색 API의 정확도순 결과 배열 순번이다. 광고 제외 실화면 오가닉 순위 또는 40개 단위 페이지 위치로 환산하지 않는다.
- 판매자 상품 URL은 URL에 포함된 상품ID와 판매자가 모두 일치할 때만 정확 상품으로 처리한다.
- 원부 카탈로그 URL을 명시적으로 입력한 경우에만 원부 순위 모드를 사용하며, 같은 원부라도 다른 상품ID 또는 다른 판매자는 정확 상품으로 승격하지 않는다.
- 30일 순위 추적은 30일 후 중단하지 않는다. 계속 수집하고 화면에는 최근 30일만 보여준다.
- 자동 갱신 기준은 매일 오전 9시와 오후 3시다.
- 자동 갱신 누락 방지를 위해 GitHub Actions와 Vercel 백업 Cron을 함께 점검한다.
- 선택 이미지 저장, 그룹 보기, 그룹 생성 및 이동, 그룹 해제는 기존 추적 데이터가 날아가지 않도록 보존 우선으로 수정한다.
