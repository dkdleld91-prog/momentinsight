# Project Memory

## 제품 구조

- 화면 원본: `src/pages/home.html`, `src/pages/admin.html`, `src/pages/client.html`
- 서버 API: `src/server/handlers`
- 배포 API 진입점: `api`
- 운영 데이터: Supabase와 서버 측 권한 검증을 사용한다.
- 배포 대상: Vercel Production `https://insight.momentlabs.co.kr`

## 고정 권한 기준

- 총관리자 계정은 `mml93-a01`이다.
- 운영팀은 연결된 광고주 1명의 데이터만 관리한다.
- 광고주는 본인 코드에 연결된 공개 데이터만 확인한다.
- 내부 메모와 광고주 공개 코멘트는 분리한다.
- 화면 숨김만으로 권한을 판단하지 않고 서버와 DB 범위를 함께 검증한다.

## 네이버 상품 순위 고정 기준

- 순위는 네이버 쇼핑 검색 API 응답의 광고 제외 오가닉 항목으로 계산한다.
- 판매자 상품 URL은 URL의 판매자 상품ID를 최우선 정확 일치 키로 사용한다.
- 판매자 상품 URL을 제목, 카테고리 또는 알려진 별칭만으로 원부 카탈로그에 자동 연결하지 않는다.
- 같은 원부라도 상품ID 또는 판매자가 다르면 정확 상품으로 처리하지 않는다.
- 원부 카탈로그 URL이나 명시적 원부 ID가 입력된 경우에만 원부 모드를 사용한다.
- 판매자 상품 조회에서는 정확 상품 순위와 관련 원부 노출을 분리한다. 관련 원부는 키워드, 브랜드·제조사·판매처 중 하나, 상위 카테고리 2단계가 모두 맞는 오가닉 원부 중 최상위 1건만 표시한다.
- 관련 원부는 화면 참고 정보이며 정확 상품 순위와 `targetProductId`를 절대 변경하지 않는다.
- 공식 API가 광고 목록과 광고 위치를 제공하지 않으므로 광고 순위나 광고 개수를 임의 계산하지 않는다.
- 단순히 판매처만 같은 다른 상품은 상품 노출 카드에서 제외한다.
- 30일 추적의 대표 상품ID는 저장된 URL에서 다시 추출해 오래된 카탈로그 ID로 오염되지 않게 한다.

## 보호 기능

- 키워드 조회, 네이버 SEO 확인, 네이버 상품 단건 순위
- 네이버 상품 30일 순위, 네이버 플레이스 30일 순위
- 오전 9시·오후 3시 자동 갱신과 최근 30일 기록
- 관리자·운영팀·광고주 로그인과 데이터 격리
- 운영 원본, 보고서 공개, 일정과 액션 플랜

## 검증 원칙

현황 확인 → 관련 코드 분석 → 구현 → 자동 테스트 → 실조회 → 데스크톱·모바일 UI 검수 → 빌드 → 문서 갱신 → 커밋 → 승인 시 배포 순서를 지킨다.
