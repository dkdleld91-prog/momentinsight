# Decisions

## 2026-07-14 · 네이버 상품 정확 일치 기준

### 판매자 상품을 원부로 자동 승격하지 않는다

- 원인: 판매자 상품 URL `5145848584`가 별칭·메타데이터 추론으로 원부 카탈로그 `59606749556`에 자동 연결되어 다른 상품 순위가 대상 순위로 표시됐다.
- 결정: 판매자 상품 URL은 URL에 포함된 판매자 상품ID를 정확 일치 기준으로 사용한다.
- 결과: 같은 원부, 다른 상품ID, 다른 판매자, 광고 상품은 정확 대상에 포함되지 않는다.

### 원부 모드는 명시 입력에만 사용한다

- 카탈로그 URL 또는 명시적 카탈로그 ID를 입력한 경우에만 원부 단위 매칭을 허용한다.
- 상품명·카테고리 유사성은 정확 ID를 대체하지 않는다.

### 동일 판매처 결과는 참고 정보로 분리한다

- 정확 대상까지 확인한 오가닉 결과 중 같은 판매처 상품을 별도 목록으로 표시한다.
- 정확 대상은 `정확 상품`, 같은 판매처의 다른 상품은 `동일 판매처`로 표시한다.

### 광고 정보는 임의 생성하지 않는다

- 네이버 공식 쇼핑 검색 API는 광고 목록과 광고 위치를 제공하지 않는다.
- 화면에 `광고 데이터 API 미제공`과 설명을 노출하고 광고 개수·광고 순위를 추정하지 않는다.

### 회귀 방지 기준을 자동화한다

- 판매자 URL이 오래된 카탈로그 ID를 무시하는지 검사한다.
- 같은 원부의 다른 상품, 다른 판매자, 광고 항목이 정확 상품으로 집계되지 않는지 검사한다.
- 관리자·광고주 화면에 정확 상품과 동일 판매처 구분이 모두 존재하는지 릴리즈 기준선에서 검사한다.
