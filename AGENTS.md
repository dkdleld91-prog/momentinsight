# AGENTS.md

이 프로젝트는 마케팅 대행사용 B2B SaaS 대시보드입니다. 모든 개발과 문서화는 광고주별 데이터 격리, 관리자 운영 효율, 대시보드 가독성, 실행 인사이트 제공을 우선합니다.

## 제품 원칙

- 광고주는 본인 브랜드 데이터만 볼 수 있어야 한다.
- 관리자는 전체 광고주, 브랜드, KPI, 보고서, 일정, 파일을 통합 관리할 수 있어야 한다.
- 내부 메모와 광고주 공개 코멘트는 데이터 구조와 UI에서 반드시 분리한다.
- 대시보드는 숫자 나열이 아니라 `현재 상태 요약 -> 핵심 지표 -> 채널별 성과 -> 인사이트 -> 다음 액션` 흐름을 따른다.
- 1차 MVP는 수동 입력과 업로드 기반의 운영 가능성을 우선한다.

## UI 원칙

- 전문적인 B2B SaaS 톤을 유지한다.
- 컬러는 블랙, 화이트, 그레이, 딥네이비를 중심으로 사용한다.
- 핵심 수치는 크게, 상세 데이터는 표로 정돈한다.
- 상태 배지, 진행률 바, 카드, 차트, 테이블을 일관되게 사용한다.
- 광고주 화면은 설명 없이도 현재 성과와 다음 할 일을 이해할 수 있어야 한다.

## 개발 원칙

- 기능은 광고주 워크스페이스 단위로 설계한다.
- 모든 주요 테이블은 `client_id` 또는 브랜드 스코프를 명확히 가진다.
- 권한 체크는 화면 숨김만으로 처리하지 않고 서버 데이터 접근 단계에서 검증한다.
- 보고서, 파일, 링크는 같은 자료 관리 모델에서 다룬다.
- 수치 계산 로직은 UI 컴포넌트에 직접 흩뿌리지 않고 도메인 유틸 또는 서버 계층에 둔다.
