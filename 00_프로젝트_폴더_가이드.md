# 모먼트 인사이트 개발 폴더 가이드

이 폴더가 앞으로 사용할 단일 작업 폴더입니다.

## 바로 쓰는 최신 작업 폴더

- `src/pages`: 실제 배포되는 홈페이지형, 운영팀형, 광고주형 화면 원본
- `03_운영시트_템플릿`: 운영시트와 엑셀 템플릿 생성/결과물
- `05_네이버_API_연동`: 네이버 검색 기반 키워드 API 키와 운영 환경변수 가이드
- `06_Supabase_연동`: Supabase 연결값, 점검 스크립트, 아임웹 연결 스니펫
- `src/server`: Vercel/Supabase 연동용 API 서버 코드
- `supabase`: Supabase 함수/설정 파일
- `docs`: 제품 기획, 화면 설계, 데이터 모델, MVP 로드맵 문서
- `database`: 데이터베이스 초안 문서

## 건드리지 않아도 되는 폴더

- `node_modules`: npm 설치 파일입니다. 삭제해도 재설치할 수 있지만 현재 서버 실행과 점검에 필요합니다.
- `.git`: 버전 관리 폴더입니다.
- `.vercel`: Vercel 프로젝트 연결 정보입니다. 배포할 때 필요합니다.

## 운영 기준

새 작업물은 가능하면 최신 작업 폴더에만 추가합니다.
화면 수정은 `src/pages`, 서버/API 수정은 `src/server`, DB 수정은 `supabase/migrations`, 운영 문서는 `docs`에만 둡니다.
이전 보관본, 영상 산출물, 빌드 결과물, CLI 캐시는 Git에 올리지 않고 필요할 때만 다시 생성합니다.

## 정리 명령

- `npm run clean:workspace:dry`: 삭제될 로컬 산출물 확인
- `npm run clean:workspace`: `dist`, `supabase/.temp` 등 재생성 가능한 파일 정리
- `npm run clean:workspace:deps`: 루트 `node_modules`까지 정리할 때만 사용

API 키가 들어갈 수 있는 `.env.local` 파일은 정리 명령에서 삭제하지 않습니다.
Vercel 로컬 연결 정보인 `.vercel`도 기본 정리 대상에서 제외합니다.

## 작업명세와 배포 원칙

- 개발 작업은 `docs/08-work-spec-autosave.md`에 기록합니다.
- `npm run work:autosave`로 작업명세서의 자동 저장 상태를 갱신합니다.
- 개발 완료 후에는 로컬 커밋까지만 진행합니다.
- `git push`, Vercel 배포, Supabase 배포는 대표님이 명시적으로 지시할 때만 진행합니다.
