# Decisions

## 2026-07-14 · 운영 수치와 광고주 빈 상태를 실제 데이터 기준으로 표시

### 실집계 전 숫자와 예시 브랜드를 운영 현황으로 사용하지 않는다

- 운영 홈의 정적 `12개·4건·3건·2건`과 브랜드 예시 목록은 실제 계정 상태처럼 오해될 수 있어 제거한다.
- 실집계 API가 연결되기 전에는 계정 연결, 운영 입력, 공개 승인, 보고서 관리로 이동하는 작업 카드를 제공한다.
- 오래된 업데이트 날짜와 담당자 예시 대신 공개 저장 시 기록되는 원칙과 현재 접속 권한 범위를 표시한다.

### 광고주 빈 상태와 공개 완료 상태를 분리한다

- 매출·ROAS·광고비·구매수·달성률이 모두 비어 있으면 `공개 준비 중`, `공개 데이터 연결 대기`, `운영팀 공개 입력 전`으로 표시한다.
- 공개 데이터가 있으면 기존 `업데이트 완료`, `운영팀 검수 완료`, `공개용` 상태를 유지한다.
- 보고서 렌더링은 저장소를 다시 읽지 않고 현재 상태의 `reportCenterSynced`를 직접 사용해 빈 상태에 데모 보고서가 혼입되는 경합을 막는다.
- 모바일 빈 상태는 중복 판단 영역을 숨기고 핵심 지표와 두 개의 액션만 남겨 카드 높이를 줄인다.

### 역할 공통 메뉴 전환은 화면 상단에서 시작한다

- 운영팀과 광고주 모두 메뉴 클릭 후 스크롤을 0으로 복귀시켜 새 화면이 중간부터 열리지 않게 한다.
- 릴리즈 기준선에서 가짜 운영 집계 제거, 빈 상태 마커, 보고서 상태 전달, 공통 스크롤 복귀를 검사한다.

## 2026-07-14 · 판매자 상품번호와 API 상품번호 분리

### 판매자 상품은 결과 링크의 상품번호로만 확정한다

- 네이버 쇼핑 검색 API의 `productId`는 검색 항목 식별자이고 판매자 상품 페이지 번호와 다를 수 있다.
- 판매자 상품 URL 조회에서는 API `productId`가 입력 번호와 우연히 같아도 정확 상품으로 처리하지 않는다.
- 결과 링크의 `/products/{id}` 또는 정규 URL이 입력 대상과 일치할 때만 `seller_link_product_id` 근거로 확정한다.
- 명시적 원부 조회는 별도 `catalog_id` 근거를 사용해 판매자 상품 판정과 섞지 않는다.

### 첫 일치 뒤에도 요청 범위를 끝까지 확인한다

- 300위 조회에서 정확 상품을 173위에 찾았더라도 300개 오가닉 슬롯까지 계속 수집한다.
- 공식 API 제공 순서를 순위 근거로 유지하며 임의 중복 제거로 순번을 재계산하지 않는다.
- 정확 상품 카드의 상세 링크는 입력한 원본 상품 URL로 고정하고 API가 반환한 링크는 검증 근거로만 보관한다.

## 2026-07-14 · 오가닉 노출 카드 밀도와 링크 역할

### 큰 결과 카드를 컴팩트 프리미엄 카드로 줄인다

- 관리자와 광고주 화면의 노출 보드는 14px 라운드, 낮은 그림자, 84px 이미지, 최소 높이 116px를 공통 기준으로 사용한다.
- 제목·순위·가격·판매처의 정보 계층은 유지하되 여백과 글자 크기를 줄여 한 화면에서 관련 원부와 정확 상품을 빠르게 비교할 수 있게 한다.
- 관련 원부 배지는 갈색 계열을 제거하고 네이버 톤에 맞는 옅은 민트 배경과 그린 글자를 사용한다.

### 상품명과 상품 열기의 목적지를 분리한다

- 상품명은 현재 조회 키워드의 네이버 쇼핑 검색 결과를 새 창으로 연다.
- `상품 열기`는 기존처럼 각 카드의 실제 상품 상세 URL을 새 창으로 연다.
- 관리자·광고주 양쪽에서 두 링크가 바뀌거나 합쳐지지 않도록 릴리즈 기준선에서 각각 검사한다.

## 2026-07-14 · 운영팀·광고주 사이드바 공통 디자인

### 운영팀 정보 구조를 공통 기준으로 사용한다

- 광고주 메뉴도 `운영`, `키워드·SEO`, `순위 조회·추적`, `광고 조사` 순서로 정리한다.
- 광고주 전용 기능과 운영팀 전용 기능은 섞지 않고, 같은 메뉴 분류 안에서 역할별 허용 항목만 표시한다.

### 두 역할의 시각 체계를 같은 프리미엄 쉘로 고정한다

- 236px 사이드바, 딥네이비 그라데이션 마크, 은은한 패널 그림자, 10px 메뉴 라운드, 활성 메뉴 왼쪽 네이비 라인을 공통 적용한다.
- 과한 색상과 장식을 피하고 화이트·그레이·딥네이비 중심으로 정리한다.
- 모바일에서는 역할별 메뉴를 같은 순서의 가로 스크롤 칩으로 제공한다.

### 회귀 검사를 추가한다

- 관리자·광고주 페이지 모두 `premium-sidebar` 공통 쉘을 포함해야 릴리즈 기준선이 통과한다.
- 광고주 메뉴 분류와 순서를 운영팀 기준으로 검사해 이전의 `성과 관리/마케팅 도구/계정 연동` 구조로 되돌아가지 않게 한다.

## 2026-07-14 · 연속 개발·커밋·충돌 방지 운영 계약

### 작업명세를 모든 변경의 기준점으로 사용한다

- 작업 시작 전 Git 상태, 최근 커밋, 현재 작업명세, 작업 상태와 다음 작업을 확인한다.
- `npm run work:autosave` 후 작업 항목을 만들고 목적, 범위, 검증, 배포 상태를 완료 시점까지 갱신한다.
- 이전 작업은 삭제하거나 다시 작성하지 않고 누적 이력으로 보존한다.

### 중복 구현과 변경 충돌을 사전에 차단한다

- 새 구현 전에 코드·문서·테스트를 검색해 기존 기능을 확장 또는 재사용한다.
- 사용자의 미커밋 변경과 관련 없는 diff를 되돌리지 않는다.
- `admin.html` 또는 `client.html`을 수정하면 공통 기능과 역할별 차이를 두 화면에서 함께 검증한다.

### 검증된 논리 단위만 커밋한다

- 대상 테스트, 다른 관점의 2차 검증, `git diff --check`, 최종 diff·상태 확인을 통과한 변경만 한 작업 단위로 커밋한다.
- 완료·보류·외부 차단 근거를 운영 문서에 동기화해 다음 작업이 같은 원인을 다시 조사하지 않게 한다.
- push와 배포는 현재 승인 범위와 필수 검증이 모두 충족된 경우에만 진행한다.

## 2026-07-14 · 플레이스 부분조회와 외부 한도 실패 기준

### 확인한 범위만 순위 근거로 사용한다

- 300개를 모두 확인하지 못한 미발견 결과는 `300위 이내 없음`으로 확정하지 않는다.
- 실제 `checkedCount`가 54라면 화면과 기록에 `54위까지 확인 · 이후 미검증`으로 표시한다.
- 외부 공급자가 명시한 오가닉 순번 필드를 우선 정렬하고 광고·중복 제거 후 화면 순위를 연속 번호로 재계산한다.

### 계정 공통 한도 오류는 자체 수집기로 전환한다

- `Monthly usage hard limit exceeded`는 Actor 개별 오류가 아니라 같은 Apify 계정의 공통 오류다.
- 첫 Actor에서 이 오류를 확인하면 나머지 Actor 호출을 중단해 요청 시간을 보존하고 자체 네이버 지도 수집기로 전환한다.
- 자체 수집기도 300개를 채우지 못하면 성공처럼 과장하지 않고 부분 상태로 저장한다.

## 2026-07-14 · 네이버 상품 정확 일치 기준

### 판매자 상품을 원부로 자동 승격하지 않는다

- 원인: 판매자 상품 URL `5145848584`가 별칭·메타데이터 추론으로 원부 카탈로그 `59606749556`에 자동 연결되어 다른 상품 순위가 대상 순위로 표시됐다.
- 결정: 판매자 상품 URL은 URL에 포함된 판매자 상품ID를 정확 일치 기준으로 사용한다.
- 결과: 같은 원부, 다른 상품ID, 다른 판매자, 광고 상품은 정확 대상에 포함되지 않는다.

### 원부 모드는 명시 입력에만 사용한다

- 카탈로그 URL 또는 명시적 카탈로그 ID를 입력한 경우에만 원부 단위 매칭을 허용한다.
- 상품명·카테고리 유사성은 정확 ID를 대체하지 않는다.

### 동일 판매처 결과는 참고 정보로 분리한다

- 정확 대상까지 확인한 오가닉 결과 중 같은 판매처 상품을 별도 목록으로 표시한다.
- 정확 대상은 `정확 상품`, 같은 판매처의 다른 상품은 `동일 판매처`로 표시한다.

### 광고 정보는 임의 생성하지 않는다

- 네이버 공식 쇼핑 검색 API는 광고 목록과 광고 위치를 제공하지 않는다.
- 화면에 `광고 데이터 API 미제공`과 설명을 노출하고 광고 개수·광고 순위를 추정하지 않는다.

### 회귀 방지 기준을 자동화한다

- 판매자 URL이 오래된 카탈로그 ID를 무시하는지 검사한다.
- 같은 원부의 다른 상품, 다른 판매자, 광고 항목이 정확 상품으로 집계되지 않는지 검사한다.
- 관리자·광고주 화면에 정확 상품과 동일 판매처 구분이 모두 존재하는지 릴리즈 기준선에서 검사한다.

## 2026-07-14 · 정확 상품과 관련 원부 노출 분리

### 순위 판정과 노출 참고 정보를 분리한다

- 정확 순위는 계속 판매자 상품ID `5145848584`의 오가닉 위치만 사용한다.
- 관련 원부는 `productExposureItems`의 `관련 원부` 항목으로만 반환하고 정확 순위, 대상 ID, 대상 모드를 바꾸지 않는다.
- 관련 원부 후보는 원부형 상품이면서 조회 키워드가 양쪽 상품명에 존재하고, 브랜드·제조사·판매처 중 하나와 상위 카테고리 2단계가 일치해야 한다.
- 여러 후보가 있으면 현재 검색 결과에서 가장 높은 오가닉 원부 1건만 표시한다.
- 단순히 판매처만 같은 다른 상품과 카테고리 또는 브랜드가 다른 원부는 카드에서 제외한다.

### 결과 화면은 프리미엄 카드로 통일한다

- 관리자와 광고주 단건 순위 화면에 동일한 카드 컴포넌트를 사용한다.
- 카드에는 실제 API가 제공하는 이미지, 상품명, 오가닉 순위, 페이지 위치, 가격, 판매처, 상품 링크만 표시한다.
- 리뷰, 평점, 예상 판매량, 광고 위치는 확인 가능한 소스가 없으므로 생성하지 않는다.
- 광고는 `광고상품 미연결`로 명확히 표시하고 별도 데이터 공급자 연동 전까지 순위와 개수를 노출하지 않는다.

### 회귀 검사를 확장한다

- 관련 원부와 정확 상품이 동시에 반환되고 순위 판정은 정확 상품에 남는지 검사한다.
- 카테고리 불일치와 브랜드 불일치 후보가 관련 원부로 들어오지 않는지 검사한다.
- 관리자·광고주 페이지의 프리미엄 카드 마커와 광고 미연결 안내를 릴리즈 기준선에 포함한다.
